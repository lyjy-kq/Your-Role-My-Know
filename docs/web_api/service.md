### 1. **获取建议**

根据用户输入的文本生成相应的建议或响应。

- **URL**: `/advice`

- **方法**: `POST`

- **描述**: 接收用户输入的文本，调用模型生成建议或响应。如果文本为空，则返回错误信息。如果生成成功，则返回匹配结果；如果文本无效或没有匹配结果，返回错误信息。

- 请求参数

  - 请求体

     (JSON):

    - **text** (字符串, 必填): 用户输入的文本。

- 请求示例

  ```
  curl -X POST "http://{服务器地址}/advice" \
  -H "Content-Type: application/json" \
  -d '{
    "id": 1,
    "text": "1.参与部门业务线软件项目和项目的售前支持工作2.撰写相关技术方案文档； 3.为技术解决方案提供参考意见."
}'
  ```

- 响应

  - 成功响应

    - **状态码**: 200

    - 响应体

      ```
    {
	    "code": 200,
	    "msg": "success",
	    "data": "{\"require\":\"[0, 0, 1, 0, 1]\",\"rate\":\"[97.64, 0.82, 0.76, 0.3, 0.28]\",\"skill\":\"[0, 1, 1, 0, 0]\",\"status\":\"success\",\"result\":[{\"businessNature\":\"国企 \",\"company\":\"北京星立方科技发展股份有限公司\",\"detailPage\":\"http://jobs.zhaopin.com/CC599175920J40071698807.htm?refcode=4019&srccode=401901&preactionid=9a221a2d-695b-466f-90a7-2be4cd69af46\",\"experience\":\"3-5年\",\"id\":1,\"jobDescription\":\"    1. 参与部门业务线软件项目和项目的售前支持工作  2. 撰写相关技术方案文档；  3. 为技术解决方案提供参考意见  4. 负责部门业务线软件项目和产品的需求调研和需求分析工作：  5. 负责对客户进行需求调研，保持良好的客户关系；6. 合适的时候承担业务线内实施项目的项目经理职务，承担项目经理的责任  职位福利：五险一金、周末双休、带薪年假  \",\"location\":\"北京-朝阳\",\"position\":\"需求分析工程师、售前工程师\",\"qualification\":\"大专\",\"salary\":\"1.3万-1.8万\"]}
    }
      ```

  - 错误响应

    - 文本为空错误

      - **状态码**: 400 

      - 响应体

        ```
        {
          "code": 400,
          "message": "文本不能为空"
        }
        ```

    - 无效消息错误

      - **状态码**: 400 Bad Request

      - 响应体

        ```
        {
          "code": 400,
          "message": "消息无效或无匹配结果"
        }
        ```

    - 超时错误

      - **状态码**: 500 Time Out

      - 响应体

        ```
        {
          "code": 500,
          "message": "处理超时，请重试"
        }
        ```

### 2. **生成图表**

- **方法**: `POST`

### 4. **获取公司描述**

根据公司类型获取对应的描述信息。

- **URL**: `/companydescription`

- **方法**: `POST`

- **描述**: 根据公司类型 `type` 获取对应的描述信息。

- 请求参数

  - **`type`** (整型, 必填): 统计的数据的类型。

- 请求示例

  ```
  
  curl -X POST "http://{服务器地址}/companydescription?type=1"
  -H: "Content-Type: application/x-www-form-urlencoded"\
  ```

- 响应

  - 成功响应

    - **状态码**: 200 OK

    - 响应体

      ```
      {
        "code": 200,
        "message": "success",
        "data": [
          {
            "sentenceId": 1,
            "content": "公司描述1"
          },
          {
            "sentenceId": 2,
            "content": "公司描述2"
          }
        ]
      }
      ```

  - 错误响应

    - **状态码**: 400 Bad Request

    - 响应体

      ```
      {
        "code": 400,
        "message": "数据返回失败"
      }
      ```
